Overview

Inequality of Opportunity (IOp) estimates are notoriously sensitive to methodological decisions: which circumstances to include, which income measure to use, which inequality index, which estimation method. A single paper can report IOp shares ranging from 10% to 50% depending on these choices.

autoresearch-iop automates the exploration of that specification space, producing a specification curve that transparently shows how IOp estimates vary across all valid methodological combinations. Instead of cherry-picking one specification, we map the entire multiverse of estimates.

This pipeline is adapted from Karpathy's autoresearch for social science research. The key insight: instead of optimizing a single metric (lower BPB = better), we explore and log every valid specification because in IOp analysis there is no single "correct" answer.

Specification Space

Each experiment is a point in a multi-dimensional space of methodological choices:

DimensionOptionsExamples
Circumstances 10 variables father_education, ethnicity, skin_tone, region_14, ...
Income 4 variants hh_pc_imputed, hh_total_reported, + log transforms
Inequality measure 7 Gini, MLD, Theil-T, Var(logs), Atkinson(ε=0.5, 1, 2)
Estimation method 5 OLS, Decision Tree, Conditional Forest, XGBoost, Random Forest
Decomposition 4 ex_ante, ex_post, lower_bound, upper_bound
Sample filter 16 all, male, female, cohorts (25-34, ..., 55-64), urban/rural × gender

Full combinatorial space is astronomically large. The agent strategically explores ~200-500 specifications following a 60/30/10 rule: 60% systematic coverage, 30% hypothesis-driven, 10% robustness checks.

Pipeline Flow (per iteration)

  1. Read previous results — Load experiment_log.jsonl and check the coverage tracker to identify which regions of the specification space remain unexplored.
  2. Identify gaps or hypotheses — The agent determines what's missing (systematic coverage) or formulates a research question to test (e.g., "Does including skin tone change IOp?").
  3. Write 5-10 specs in analyze.py — Each ExperimentSpec defines one point in the multiverse, with a clear rationale documenting why this combination was chosen.
  4. Execute python run_experiment.py — The fixed harness validates each spec, loads the data, runs the estimation method, computes IOp with bootstrap confidence intervals, runs diagnostics, and appends results to the log.
  5. Review results and diagnostics — Check for flags (overfitting, small samples, wide CIs) and adjust strategy accordingly.
  6. Every ~25 experiments: generate synthesis — Specification curve plot, summary tables, and figures for publication.
  7. Repeat until coverage exceeds 80% of the target space or the researcher stops the loop.

3. Inequality Measures — core/inequality_measures.py

Seven inequality measures implemented from scratch with optional sample weights:

G Gini Coefficient

Covariance formula: G = 2·cov(y, F(y)) / μ. Handles weighted data and ties.

M MLD / GE(0)

Mean Log Deviation: log(μ) - mean(log(y)). Exactly additively decomposable.

T Theil-T / GE(1)

Theil index: mean((y/μ)·log(y/μ)). Also additively decomposable.

V Variance of Logs

Var(log(y)). Classic measure, but not decomposition-consistent.

A Atkinson(ε)

Normative index with aversion parameter. Three presets: ε = 0.5, 1.0, 2.0.

4. IOp Decomposition — core/decomposition.py

Four decomposition approaches, each giving a different interpretation of IOp:

TypeFormulaInterpretation
lower_bound I(ŷ) / I(y) Parametric (Ferreira-Gignoux). IOp of predicted values over total. Conservative.
ex_ante I(μk) / I(y) Between-type inequality over total. Uses group means.
ex_post 1 − Σ wk·I(yk) / I(y) Total minus weighted within-type inequality. Exact for GE measures.
upper_bound Same as ex_post Explicitly labeled upper bound for clarity.

5. Data Loader — core/data_loader.py

The DataRegistry class provides validated access to ESRU-EMOVI 2023 data. It handles 16 sample filters (gender, age cohorts, urban/rural combinations), validates that requested circumstances and income variables exist in the data, and returns aligned (y, X) arrays ready for estimation.

6. Experiment Harness — run_experiment.py

The fixed harness that the agent never modifies. It orchestrates the full pipeline: 

run_single_experiment(spec):
    1. Validate spec (domain rules)
    2. Load data + apply sample filter
    3. Handle income transformations (log detection)
    4. Dispatch to estimation method (OLS/Tree/XGB/RF)
    5. Compute IOp with bootstrap CI (200 iterations)
    6. Run diagnostics (flags, warnings)
    7. Build ExperimentResult
    8. Append to log (JSONL + TSV)

7. Bootstrap CI — evaluation/metrics.py

Percentile bootstrap with configurable iterations (default 200) and confidence level (95%). Each bootstrap iteration re-estimates the full pipeline (re-fit model + recompute IOp) on a resampled dataset, producing a distribution of IOp shares. CIs come from the 2.5th and 97.5th percentiles.

8. Experiment Log — orchestration/experiment_log.py

Append-only JSONL log. Every experiment (success, failure, or invalid) gets recorded with its full specification, results, diagnostics, and runtime. A parallel TSV mirror provides human-readable access. Results are never deleted or modified.

Estimation Methods

OLS Parametric (Ferreira-Gignoux)

methods/parametric.py

OLS regression of income on dummy-encoded circumstances. R² provides the parametric lower bound. I(ŷ)/I(y) gives measure-specific IOp share. Only valid decomposition: lower_bound.

DT Decision Tree

methods/nonparametric.py

sklearn DecisionTreeRegressor as proxy for ctree. Partitions individuals into "types" (leaf nodes). Type means used for ex-ante; within-type inequality for ex-post. Pruned via ccp_alpha.

XGB XGBoost

methods/ml_methods.py

Gradient boosting with 5-fold cross-validated predictions to avoid overfitting bias. SHAP values for variable importance. Only valid decomposition: ex_ante.

RF Random Forest

methods/ml_methods.py

Ensemble of 200 trees with cross-validated predictions. Captures non-linear circumstance-income relationships. SHAP importance available. Decomposition: ex_ante.

Methodological Safeguards

✓
Append-only log — All results are recorded and never deleted. Success, failure, and invalid specs are all logged for full transparency.
✓
Coverage-based exploration — The agent selects specifications based on what's missing from the explored space, not based on which results look "interesting". This prevents p-hacking by proxy.
✓
Spec validation — Domain rules prevent invalid combinations (e.g., double-log, OLS + ex_post, no circumstances). Invalid specs are caught before execution.
✓
Diagnostics — Sample size guards (min 100 observations), CI width checks, overfitting detection (comparing in-sample R² vs CV R²), and automatic flagging of suspicious results.
✓
Reproducibility — Deterministic seeds throughout. Same spec_id + seed = identical results. Bootstrap RNG is seeded per-experiment.
✓
Deterministic spec hashing — Each specification gets a unique 12-character SHA-256 hash. Circumstance order doesn't matter (auto-sorted). Duplicate specs are detected automatically.

Quick Start

# Clone the repository
git clone https://github.com/Lalitronico/autoresearch-iop.git
cd autoresearch-iop

# Install dependencies
pip install -e ".[dev]"

# Prepare data (synthetic for testing, or place real .dta in data/raw/)
python prepare.py --synthetic

# Run baseline experiments (3 specs defined in analyze.py)
python run_experiment.py

# Run the test suite (44 tests)
python -m pytest tests/ -v

# The agent modifies analyze.py to add new specs, then re-runs
# Results accumulate in results/experiment_log.jsonl

Data

Uses ESRU-EMOVI 2023 from Mexico (Centro de Estudios Espinosa Yglesias). 17,843 respondents aged 25-64. Place the raw .dta file in data/raw/ and run python prepare.py. For testing without real data, use python prepare.py --synthetic which generates 500 observations with realistic distributions.
